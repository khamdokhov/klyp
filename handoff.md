## 1. Продукт в одном абзаце

Приложение сидит в строке меню. Оно фоново отслеживает буфер обмена и сохраняет историю скопированного — **любой контент**: обычный текст, форматированный текст, ссылки, цвета, изображения, файлы. По глобальному хоткею или клику по маскоту открывается панель с историей: поиск, превью, источник (из какого приложения скопировано), закрепление избранного. Выбор элемента возвращает его в буфер (а при включённой опции — сразу вставляет в активное приложение). Маскот — векторный блоб, нарисованный кодом, — отражает состояние приложения мимикой и пластикой: ловит контент, полнеет по мере заполнения истории, спит при простое, просит разрешения, отворачивается когда намеренно пропускает секрет. Контраст со строкой меню и адаптацию под светлую/тёмную тему обеспечивает template-режим иконки — система красит силуэт сама.

## 2. Стек и ограничения

- **Язык:** Swift, последняя стабильная версия тулчейна.
- **UI:** SwiftUI как основной слой; AppKit там, где SwiftUI не покрывает (`NSStatusItem`, кастомная отрисовка иконки, `NSPanel` для панели истории, синтез нажатий).
- **Минимальная цель:** macOS 14.0. Обоснование: зрелый `MenuBarExtra`/`Settings`, `SMAppService`, symbol effects, современный SwiftUI и Observation.
- **Архитектура приложения:** `NSApplicationDelegateAdaptor` + AppKit-управляемый статус-айтем (нужен полный контроль над покадровой анимацией иконки), SwiftUI для всех панелей и окна настроек.
- **Зависимости:** по умолчанию **ноль внешних пакетов**. Глобальный хоткей — на Carbon `RegisterEventHotKey` в тонкой Swift-обёртке. Хранилище — `Codable` + файлы (см. раздел 5). Если для записи хоткея в UI понадобится рекордер — допустимо подключить `KeyboardShortcuts` (Sindre Sorhus) как **единственную** опциональную зависимость; иначе обойдись своим рекордером.
- **Сборка:** обычный Xcode-проект (`.xcodeproj`), схема под Release/Debug, без CocoaPods/Carthage.

## 5. Слой буфера обмена

### 5.1 Мониторинг
`NSPasteboard` не шлёт событий. Поллинг: `Timer` (или `DispatchSourceTimer`) с интервалом **0.4 с**, сравнение `NSPasteboard.general.changeCount` с запомненным. Изменился — запускаем захват. Поллер должен **уметь паузиться** (когда панель открыта — не обязателен, но не отключай без причины; главное — пауза при намеренной самозаписи, см. 5.5).

### 5.2 Захват всех представлений
Идём по `pasteboard.pasteboardItems`. Для каждого `NSPasteboardItem` сохраняем **все** доступные `types` как `[String: Data]` (`item.data(forType:)`). Это позволяет вернуть контент без потери формата. Источник: `NSWorkspace.shared.frontmostApplication` **в момент захвата** — имя + bundle id + иконка (иконку не сохраняем в стор, тянем по bundle id при показе).

### 5.3 Определение типа (`ClipItemKind`)
Приоритет при классификации: `fileURL` → `.file`; наличие `public.url` без файла → `.link`; `png`/`tiff` → `.image`; `org.nspasteboard.ColorType`/цветовые данные → `.color`; `rtf`/`rtfd`/`html` + plain → `.richText`; только `string` → `.text`. Тип влияет на иконку, превью и реакцию блоба.

### 5.4 Чувствительный контент (`SensitiveContent`)
Проверяем типы item на маркеры nspasteboard.org:
- `org.nspasteboard.TransientType` — **всегда игнорируем**, без настройки (это контракт «не сохранять»).
- `org.nspasteboard.AutoGeneratedType` — игнорируем по умолчанию; поведение управляется настройкой.
- `org.nspasteboard.ConcealedType` — секрет. По умолчанию **пропускаем**. Если пользователь включил хранение секретов — сохраняем **только в `SecureStore` (Keychain)**, с укороченным временем жизни, и помечаем в индексе как secure (тело в обычный `BlobStorage` не попадает).

### 5.5 Петля самозаписи
Когда `PasteService` сам пишет в буфер, `changeCount` вырастет. Перед записью сохраняем «ожидаемый» `changeCount`+1 и в поллере **игнорируем ровно это значение**, чтобы не словить собственную вставку как новый элемент.

### 5.6 Дедуп
Перед вставкой нового элемента сравниваем с последним (и, опц., ищем совпадение в истории по хешу содержимого). Совпало — **не дублируем, поднимаем существующий наверх** и обновляем `timestamp`. Хеш считаем по нормализованному «ключевому» представлению (для текста — строка; для картинки — хеш data).

## 6. Модель данных

```
ClipItem
  id: UUID
  createdAt: Date
  kind: ClipItemKind          // text, richText, link, color, image, file
  representations: [String: Data]   // НЕ для secure-элементов
  previewText: String?        // для текстовых/ссылок/цвет hex
  thumbnailRef: String?       // имя файла миниатюры для image/file
  source: SourceApp?          // name, bundleIdentifier
  isPinned: Bool
  isSecure: Bool              // тело в Keychain, не в representations
  byteSize: Int               // для лимита по размеру
```

`ClipItem` — value type. В индексе на диске лежат **только метаданные** (`id, createdAt, kind, previewText, thumbnailRef, source, isPinned, isSecure, byteSize`). Сами `representations`/миниатюры/secure-тела — отдельно (см. 5.4, раздел 7).

## 7. Хранилище и вытеснение

- **Индекс** (`HistoryIndex`): один `Codable`-файл в `Application Support/Blobboard/index.json` (атомарная запись). Держит порядок и метаданные.
- **Тела** (`BlobStorage`): `Application Support/Blobboard/items/<id>.dat` (+ `<id>.thumb.png` для миниатюр). Ленивая загрузка тела только при вставке/превью.
- **Secure-тела** (`SecureStore`): в Keychain по ключу `id`.
- **Eviction** (`EvictionPolicy`) — комбинированный, срабатывает после каждого добавления и при изменении лимитов:
  - по количеству (`maxItems`, диапазон 50–500, дефолт 200),
  - по суммарному размеру на диске (`maxBytes`, дефолт ~500 МБ),
  - по возрасту (`maxAge`, дефолт 30 дней; 0 = без ограничения).
  - **Закреплённые (`isPinned`) исключены из вытеснения** всегда.
  - Стратегия — LRU/по `createdAt`. Удаление элемента чистит и индекс, и файлы тела/миниатюры, и Keychain.
- При уменьшении лимита в настройках — немедленный прогон eviction.

## 8. Возврат в буфер и авто-вставка

`PasteService` два уровня:

1. **Copy-back (база, без разрешений):** `clearContents()` → записать **все** `representations` элемента обратно в `NSPasteboard.general`. Пользователь жмёт ⌘V сам. Для secure-элемента тело берётся из Keychain.
2. **Auto-paste (опция, требует Accessibility):**
   - Перед открытием панели запомнить прежнее `frontmostApplication` (`FocusRestorer`), т.к. панель заберёт фокус.
   - После выбора: восстановить буфер → `app.activate()` прежнего приложения → `KeystrokeSynthesizer` синтезирует ⌘V через `CGEvent` (keyDown/keyUp, virtual key V + `.maskCommand`).
   - Доступность проверяется через `AXIsProcessTrusted()`. Если права нет — авто-вставка не выполняется, тогл в настройках показывает «нужно разрешение» (см. 10).

Battery-aware: ничего не синтезируем, если приложение не имеет фокусной цели (нет прежнего frontmost) — тихо падаем в copy-back.

## 9. Панель истории и хоткей

- **Вызов:** глобальный хоткей (дефолт ⇧⌘V) и клик по маскоту. Хоткей — `GlobalHotkey` на Carbon `RegisterEventHotKey` + `InstallEventHandler`, в Swift-обёртке с колбэком; перерегистрируется при смене в настройках.
- **Панель:** `NSPanel` (`.nonactivatingPanel` где уместно, но **с возможностью стать key** — иначе не наберёшь текст в поиске), хостит SwiftUI `HistoryView`. Фон — системный материал (`NSVisualEffectView`, `.popover`/`.menu`), скруглённые углы, тень — как у нативного поповера. Появляется у иконки.
- **`HistoryView`:**
  - поле поиска сверху (фильтр по `previewText`, имени источника, типу),
  - список (`ClipRowView`): иконка типа (SF Symbol) + иконка приложения-источника + превью (текст — 1–2 строки; image — миниатюра; color — свотч + hex; file — имя + иконка файла) + относительное время,
  - действия на строке: вставить (Enter/клик), закрепить (пин), удалить;
  - футер: счётчик, «Clear all», шестерёнка (открыть настройки).
  - **Клавиатура:** ↑/↓ навигация, Enter — вставить, ⌫ — удалить, ⌘F — фокус в поиск, Esc — закрыть. Полная управляемость без мыши обязательна.
  - **Пустое состояние:** `EmptyStateView` со спящим блобом.

## 10. Настройки (`Settings` scene, TabView)

Нативное окно настроек в стиле System Settings. Бэкенд — `AppSettings` (`@Observable`, поверх `UserDefaults`).

**General**
- Launch at login — тогл, **завязан на `SMAppService.mainApp`**: `register()`/`unregister()`, а текущее состояние читается из `.status`, не из своего флага (иначе соврёт, если пользователь выключил в System Settings → Login Items).
- Global shortcut — рекордер комбинации.
- Liveliness — сегмент `Full / Reactions only / Static` (управляет активностью блоба).

**History**
- Max items — слайдер 50–500.
- Max size on disk — слайдер/степпер.
- Auto-delete after — пикер возраста (Off / 7 / 30 / 90 дней).
- Какие типы захватывать — тоглы (text/richText/images/files/links). Выключение не удаляет уже сохранённое, только перестаёт ловить новое.

**Privacy**
- Store sensitive (concealed) items — тогл, **дефолт OFF**, с подписью-предупреждением, что секреты будут храниться зашифрованно в Keychain. `TransientType` не управляется и игнорируется всегда.
- Store auto-generated items — тогл, дефолт OFF.
- Ignore-list по bundle id (не захватывать из перечисленных приложений) — опционально, если успеваешь в рамках вехи M6.

**Paste**
- Paste directly into active app — тогл авто-вставки. **Не голый `Bool`:** при включении дёргает запрос Accessibility; если права нет — тогл не встаёт в ON, рядом — статус «Permission required» и кнопка, открывающая нужную панель System Settings. Состояние тогла отражает реальную выдачу права.

## 11. Стандарт качества кода (критерий приёмки, не пожелание)

Код едет в публичный GitHub. Требования:

**Язык и стиль**
- Весь код, комментарии, имена, doc-строки, коммиты, README — **на английском**.
- Имена — по Swift API Design Guidelines. Без венгерской нотации, без `mgr`/`tmp`/`data2`.
- Никаких эмодзи нигде — ни в коде, ни в коммитах, ни в README.

**Комментарии**
- Комментарий объясняет **почему**, а не **что**. Запрещены комментарии, дублирующие код (`// increment counter`, `// loop over items`).
- Публичный нетривиальный API документируется `///` там, где это правда помогает. Тривиальное — не документируется.
- **Ноль ИИ-артефактов:** ни «Here's the implementation…», ни «This elegant/robust solution…», ни оставленных `TODO/FIXME`, ни закомментированного кода, ни placeholder-ов.

**Архитектура и типобезопасность**
- Слои — за протоколами, зависимости инъектятся (тестируемость).
- Value types по умолчанию; reference types — где есть identity/мутабельное разделяемое состояние.
- Никаких stringly-typed там, где подходит enum. `ClipItemKind`, состояния блоба, типы pasteboard — типизированы.
- Concurrency: UI под `@MainActor`, структурированная многозадачность, без гонок. Поллер и захват не блокируют main лишним.

**Ошибки**
- Типизированные ошибки, осмысленный `throws`. Нет пустых `catch {}`, нет `try!`/force-unwrap вне гарантированно безопасных мест (а где есть — однострочный комментарий «почему безопасно»).

**Тесты** (XCTest), как минимум:
- захват и дедуп, eviction (все три ограничителя + исключение пинов), фильтрация sensitive-типов, round-trip хранилища, переходы `BlobStateMachine`.

**Инструменты и гигиена репозитория**
- `.swiftlint.yml` и `.swiftformat` в комплекте, код проходит их без ворнингов.
- `.gitignore` корректный (`DerivedData/`, `*.xcuserdata`, `.DS_Store`, build artifacts).
- Коммиты — Conventional Commits, осмысленные сообщения, маленькими логическими порциями (не один «init» на всё).
- `README.md`: что это, для чего, скриншот/гиф (плейсхолдер ок), требования, как собрать, какие разрешения запрашивает и зачем — **без маркетингового и ИИ-фluff**. `LICENSE` (MIT).

**Производительность/энергия (это тоже качество)**
- Анимация блоба ~30 fps, и **замедляется/замирает в ambient-простое**; не крутить таймер вхолостую на полной частоте. Учитывать Low Power Mode.
- Миниатюры — маленькие; полные данные не держим в памяти списка.

## 12. Дизайн (Apple HIG, нативный UI)

- **Только системные компоненты:** `Form`, `Toggle`, `Slider`, `Picker`, `LabeledContent`, `TabView`, `Settings`. Окно настроек выглядит как System Settings.
- **Материалы:** фон панели и поповеров — системные материалы (`.regularMaterial`/`NSVisualEffectView`), не залитые цветом прямоугольники.
- **Иконография:** только **SF Symbols**, hierarchical/palette rendering; для маскота-акцентов — symbol effects, где уместно.
- **Цвет:** только семантические цвета (`Color.primary`, `.secondary`, `Color(nsColor: .labelColor)` и т.п.). **Хардкод hex запрещён** — иначе сломается тёмная тема и Increase Contrast.
- **Типографика:** системный шрифт, корректные роли (`.body/.callout/.caption`), без произвольных кеглей.
- **Сетка/отступы:** 8pt-grid, стандартные инсеты, выравнивание по бейзлайну.
- **Анимации:** `spring`/`smooth`, сдержанно; уважать **Reduce Motion** (заменять движение на лёгкий кроссфейд).
- **Доступность:** VoiceOver-лейблы для строк и действий, полная клавиатурная навигация, фокус-ринги, Increase Contrast, Dynamic Type там, где применимо.

## 13. Маскот: блоб (ядро «живости»)

### 13.1 Геометрия капли (`BlobRenderer`)
Капля — **гладкая замкнутая кривая через N контрольных точек** (N = 10–12), расставленных по окружности, радиус каждой гуляет.

1. Точки: для `i` в `0..<N`, `θ = 2π·i/N`, `r_i = base·(1 + wobble_i)`, `p_i = center + (cos θ, sin θ)·r_i`.
2. **Сглаживание — обязательно через замкнутый Catmull-Rom → кубический Безье**, иначе будет многоугольник. Для сегмента `p_i → p_{i+1}` контрольные точки безье:
   - `c1 = p_i + (p_{i+1} − p_{i−1})·k`
   - `c2 = p_{i+1} − (p_{i+2} − p_i)·k`
   - `k ≈ 1/6` (0.16–0.2). Индексы — по модулю N (кольцо).
3. Рисуем в `NSBezierPath`/`CGPath`, замыкаем.

### 13.2 Wobble (дыхание/желе)
`wobble_i = Σ sin(t·freq_j + i·phase_j)·amp_j` по 2–3 слоям с разными `freq/phase`. Амплитуды малые (5–10% от `base`). `t` — время из цикла анимации. Один слой — ровно, два-три — органичный «живой» хаос.

### 13.3 Squash & stretch (`BlobSpring`)
Деформация всего блоба вокруг центра с **сохранением объёма**: `sx = 1 + squash`, `sy ≈ 1/sx`. `squash` и вертикальный `bob` управляются **пружинной моделью** (целевое значение + жёсткость + затухание), чтобы прыжки/шлепки пружинили, а не дёргались линейно. Именно s&s даёт «существо», а не пульсирующее пятно.

### 13.4 Мордочка (`BlobFace`)
Глаза и рот — отдельные заливки **поверх** капли, едут вместе с `bob`/`squash`. Стили глаз: `open`, `happy(^^)`, `closed(--)`, `wide(oo)`, `sleepy`, `sad`, `averted`, `hearts`, `checkmarks`, `dizzy`. Моргание — на 2 кадра подменять овал горизонтальной чертой. Рот: `neutral/smile/open/flat/wavy`.

### 13.5 Template и цикл
Рисуем **одним цветом с альфой**, `image.isTemplate = true` → система красит силуэт под панель (контраст и тема бесплатно). Цикл: таймер/`CVDisplayLink` тикает `t` → `BlobRenderer` считает кадр по текущим `BlobParameters` → печёт `NSImage` → `statusItem.button?.image`. Цветные акценты (вспышка на «поймал») — короткими transient-кадрами поверх силуэта (гибридная стратегия): форму несёт силуэт, цвет — лишь короткий акцент.

### 13.6 `BlobParameters` (крутилки одного кадра)
`base`, `wobbleAmp[]`, `wobbleFreq[]`, `squashTarget`, `bobTarget`, `tilt`, `eyeStyle`, `mouthStyle`, `accentColor?`, `accessory` (`none/zzz/question/bang/sparkle/sweat/folder/page/heart`), `blinkRate`. Состояние = пресет этих значений; `render()` не меняется при добавлении состояний.

### 13.7 Состояния (`BlobState`) и автомат (`BlobStateMachine`)
Два слоя. **Ambient** (длительный фон, одно активно) и **Transient** (короткий, 0.4–1.2 с, проигрывается поверх и возвращает в текущий ambient). События приложения переводят автомат. Уважать `Liveliness` из настроек (`Full` — всё; `Reactions only` — только transient на захват/вставку, ambient статичен; `Static` — один кадр) и Reduce Motion.

**Ambient:**
| Состояние | Когда | Параметры (суть) |
|---|---|---|
| `idle` | обычная работа | медленный wobble, лёгкий bob, глаза open, редкое моргание |
| `content` | недавно активничал | wobble чуть живее, smile |
| `stuffed` | история у лимита | `base` больше («толще»), wobble вялее |
| `sleeping` | пусто/ночь/долгий простой | сплющен (`sy<1`), wobble почти ноль, глаза `--`, `zzz` |
| `away` | нет ввода юзера давно | занят своим: медленное кручение/качание |

**Transient:**
| Состояние | Триггер | Параметры (суть) |
|---|---|---|
| `capturedText` | поймал текст | резкий squash+bob, глаза `^^`, акцент зелёный |
| `capturedImage` | поймал картинку | «вспышка» (глаза `oo`→моргание), акцент |
| `capturedFile` | поймал файл | в руках `folder` на миг |
| `capturedLink` | поймал ссылку | `sparkle`, smile |
| `capturedColor` | поймал цвет | акцент = сам цвет на миг |
| `duplicate` | дубль | глаза закатились, лёгкий вздох (медленный bob вниз) |
| `skippedSecret` | пропустил concealed при OFF | `averted`, отворот (`tilt`), прикрыл глаза |
| `pasted` | вставил выбранное | глаза `checkmarks`, довольный bob |
| `pinned`/`unpinned` | пин/снятие | `sparkle` / нейтрально |
| `evicted` | вытеснил старое | короткий «выдох», `base` чуть спадает |
| `needsPermission` | нет Accessibility при попытке авто-вставки | `tilt`, глаза `wide`, `question` |
| `error` | сбой записи/хранилища | `bang`, глаза `dizzy` |
| `searching` | открыт поиск | прищур (узкие глаза), `tilt` к курсору |

Переходы тестируются (`BlobStateMachineTests`): событие → ожидаемый transient → возврат в корректный ambient по таймеру; конфликтов (одновременно sleep и capture) быть не должно — transient всегда поверх, ambient запоминается и восстанавливается.

## 14. Приватность и безопасность

- Дефолты приватные: секреты не хранятся, auto-generated не хранятся, transient игнорируется всегда.
- Секреты (если включены) — только Keychain, не в файлах истории; короткий TTL.

- App Sandbox: авто-вставка и глобальный хоткей конфликтуют с жёстким сэндбоксом; для распространения через GitHub (не App Store) — сборка с Developer ID, **подпись + нотаризация**. Entitlements держать минимальными. В README — раздел про первый запуск и выдачу Accessibility.
